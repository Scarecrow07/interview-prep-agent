# AI 面试准备方案生成器

上传 JD（职位描述）和简历，自动生成结构化面试准备方案——纯浏览器端运行，无需后端服务器。

[立即使用](https://scarecrow07.github.io/interview-prep-agent/)

---

## 功能介绍

### 6 大报告模块

| 模块 | 内容 |
|------|------|
| **JD 核心分析** | 岗位名称、行业、核心职责、硬技能（含熟练度/是否必备）、软技能、经验/学历要求、评价关键维度 |
| **简历匹配分析** | 总体匹配度评分（0-100 环形图）、技能逐项匹配状态、核心优势、差距与风险预警、个性化准备策略 |
| **自我介绍方案** | 三版脚本（1 分钟 / 2 分钟 / 3 分钟），各有适用场景、完整脚本、关键亮点；附带表达技巧与常见错误提醒 |
| **面试问答题库** | 涵盖技术深度、项目追问、行为面试、情景题、HR 通用、动机匹配六大类别，含参考答案要点与备考建议 |
| **笔试编码题** | 技术岗位自动生成，非技术岗位跳过。含题目描述、示例、解题思路、参考代码、复杂度分析与关联 LeetCode 题 |
| **学习资料推荐** | 按差距优先级推荐书籍、课程、文档、视频等，标注难度与预计耗时；附带学习路径和重点知识要点 |

### 支持的 API 提供商

| 提供商 | 默认模型 | 特点 |
|--------|----------|------|
| **DeepSeek**（默认） | DeepSeek-V4-Flash | 性价比高，支持深度思考模式 |
| **OpenAI** | GPT-5.4 | 旗舰模型，生产级能力 |
| **Kimi（月之暗面）** | Kimi K2.6 | 超长上下文，Agent 能力突出 |
| **Qwen（阿里云百炼）** | Qwen-Plus | 丰富模型选择，长上下文支持 |
| **Doubao（火山引擎）** | Doubao Seed 2.1 Pro | 国产旗舰，多模态能力强 |

### 文件上传

| 区域 | 支持格式 | 说明 |
|------|----------|------|
| **JD 上传** | .txt / .pdf / .docx / .png / .jpg / .jpeg / .webp | 图片自动触发 LLM OCR 识别 |
| **简历上传** | .txt / .pdf / .docx | 文本提取后自动填充 |
| 上传方式 | 拖拽 / 点击 | 双模式支持 |

### 其他特性

- **纯浏览器端运行**：无需后端服务器，所有计算在浏览器内完成
- **API Key 本地保存**：每个提供商的 Key 按名字独立存储，切换自动恢复
- **响应式设计**：桌面 / 平板 / 手机三档适配
- **智能岗位识别**：自动判断技术岗/非技术岗，决定是否生成编码题
- **3 分钟超时保护**：API 请求超时自动中断，避免页面卡死
- **打印友好**：隐藏交互控件，只保留报告内容

## 如何使用

### 1. 准备 API Key

选择 API 提供商，进入该平台的 API Key 管理页面创建 Key：

- [DeepSeek](https://platform.deepseek.com/api_keys)
- [OpenAI](https://platform.openai.com/api-keys)
- [Kimi](https://platform.kimi.com/console/api-keys)
- [Qwen（阿里云百炼）](https://bailian.console.aliyun.com/)
- [Doubao（火山引擎）](https://console.volcengine.com/ark/region:ark+cn-beijing/apiKey)

### 2. 填写或上传 JD

将职位描述粘贴到左侧文本框（至少 50 字符），或通过拖拽/点击上传文件自动填充。图片格式的 JD（如截图）会自动调用 OCR 提取文字。

### 3. 填写或上传简历

将个人简历粘贴到右侧文本框，或上传文件自动填充。

### 4. 选择岗位类型

- **自动识别**：系统根据 JD 关键词自动判断
- **技术岗位**：强制生成编码题
- **非技术岗位**：跳过编码题

### 5. 生成报告

点击"生成面试准备方案"，等待 6 个模块依次生成完成。生成过程中可以实时看到每一步的进度状态。

### 6. 查看与打印

报告生成后可以滚动查阅全部 6 个模块。使用浏览器打印功能（Ctrl+P）可获取干净的报告排版。

## 技术实现

- **纯前端**：HTML + CSS + JavaScript，无后端依赖
- **API 调用**：OpenAI 兼容格式，支持 JSON 模式和视觉理解
- **文件解析**：pdf.js（PDF）、mammoth.js（DOCX）
- **数据持久化**：localStorage 保存 API Key
- **安全**：API Key 仅存储在本地，不会发送到第三方
- **部署**：GitHub Pages

